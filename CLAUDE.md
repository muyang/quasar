# 能量石 (Energy Stone) App - MVP 阶段开发需求文档

## 1. 项目背景与产品核心
本项目是一个主打“情绪治愈与仪式感”的习惯养成与社交 App（MVP 阶段暂不包含复杂社交，主攻单机体验）。
核心玩法围绕一颗虚拟的“能量石”展开：用户每日长按屏幕为石头充能；若不充能，石头能量会衰减，直至“枯竭死亡”。

## 2. 技术栈选型
* **后端架构:** Python 3.10+ / FastAPI / SQLAlchemy / SQLite (MVP) / APScheduler (定时任务)
* **前端架构:** Flutter (跨平台) / `http` 库对接 API / `vibration` 或自带 `HapticFeedback` 处理震动反馈
* **环境要求:** 务必保证依赖版本的稳定性，避免使用过于激进或处于实验阶段的底层框架。

## 3. 核心业务状态机与数值逻辑
* **初始状态:** 新石头能量为 10，状态为 `ALIVE`，枯竭次数 `death_count` 为 0。能量上限为 100。
* **充能逻辑 (Check-in):** * 调用接口，能量增加一个随机整数（1~5点，采用加权随机）。
    * 更新最后充能时间。
    * 返回充能结果和一句 AI 生成的治愈祝福语。
* **衰减逻辑 (Decay):**
    * 每日凌晨 (00:00) 触发。
    * 若某石头在计算日当天没有充能记录，则能量扣除 3 点。
    * **死亡判定:** 若扣除后能量 $\le$ 0，则 `death_count` + 1。
    * 若 `death_count` $<$ 3，能量重置为 5（保留火种）。
    * 若 `death_count` $\ge$ 3，状态变为 `DEAD`，能量清零，不可再充能。

## 4. 实施路径 (按阶段执行)

### 阶段一：后端基建与 API (FastAPI)
**开发目标：** 搭建后端基础架构，实现数据库模型与充能接口。
**具体任务：**
1.  初始化 FastAPI 项目结构，配置 `requirements.txt`。
2.  设计 SQLite 数据库模型 `EnergyStone`（包含字段：id, user_id, current_energy, death_count, status, last_charge_time）。
3.  开发 `POST /api/stone/{stone_id}/charge` 接口，实现加权随机数值增量，并处理石头是否已死亡的边界判定。
4.  编写基础单元测试，确保充能接口按预期工作。

### 阶段二：后端衰减引擎 (定时任务)
**开发目标：** 实现不可篡改的生命周期引擎。
**具体任务：**
1.  引入 `APScheduler`。
2.  编写每日零点执行的定时任务脚本，扫描所有 `status == 'ALIVE'` 的石头。
3.  严格按照“需求 3”中的【衰减逻辑】与【死亡判定】更新数据库。
4.  务必使用数据库事务 (Transaction) 包裹更新操作，确保数据一致性。

### 阶段三：前端沉浸式交互 (Flutter)
**开发目标：** 实现主界面的长按动画与 API 联调。
**具体任务：**
1.  初始化 Flutter 页面结构，设定深色治愈系主题 (`Color(0xFF121212)`)。
2.  实现**长按交互组件**:
    * 使用 `AnimationController` 实现长按 3 秒充满的逻辑。
    * 长按期间：中心光晕动态扩大，亮度增加，并伴随阶段性的轻微震动 (`HapticFeedback.lightImpact`) 模拟心跳。
    * 松开手指：若未满 3 秒，动画回退，能量消散。
3.  对接后端 `/charge` API：长按满 3 秒后，触发强烈震动，调用 API，并使用弹窗或底部面板展示后端返回的“祝福语”。

## 5. 代码质量与系统级约束（最高优先级）
作为 AI 编码助手，在执行上述任务时，**必须**遵守以下红线：

1.  **绝对的代码正确性:** 在生成任何代码文件前，务必在内部推演逻辑。**严禁输出带有语法错误、未闭合标签或缺少依赖项的代码。**
2.  **严密验证:** 若需生成图表配置（如 Mermaid）或复杂 SQL 查询，必须经过严格核查验证，确保代码无误且符合标准语法规范后方可写入文件。
3.  **最小可用原则:** 优先保证主流程（打卡、衰减）跑通，不要过度设计未提及的复杂权限或鉴权系统。
4.  **清晰的日志:** 后端定时任务和前端网络请求必须加入关键节点的 `print` 或 `logger` 记录，以便排查问题。
